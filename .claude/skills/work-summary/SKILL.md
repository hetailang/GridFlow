# work-summary

当用户要求“整理当前新功能/修改到相关文件”时，请按以下步骤执行：

### 1. 更新 README.md (变更日志)
在 `README.md` 的 **Changelog** 或 **更新日志** 部分追加记录：
* **格式规范**：采用 `日期 - 摘要` 作为标题行。
* **细节收纳**：所有具体的修改细节、技术要点及背景说明必须置于该标题下方的**缩进块**中。
* **文档同步**：全面检查 README 的“功能介绍”、“快速开始”或“配置说明”部分。若本次修改导致原有功能逻辑、API 接口或使用方式发生变化，必须同步更新相关章节，确保文档与代码版本一致。

### 2. 更新 CLAUDE.md (项目规范)
* **判断逻辑**：评估本次修改是否涉及项目架构、技术栈、命名规范或核心开发流程的变更。
* **更新触发**：
    * 若仅为 Bug 修复或 UI 调整，无需改动。
    * 若增加了新工具链、改变了目录结构或引入了新的开发约定，需更新 `CLAUDE.md` 中的指令或指南部分。

### 3. 润色要求
* 记录使用的语言应条理清晰、专业简洁。
* 确保 Markdown 层级正确，不破坏原有的文档结构。
